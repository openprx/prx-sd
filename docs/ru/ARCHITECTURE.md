# Архитектура

PRX-SD организован как рабочее пространство Cargo с модульными крейтами, каждый из которых отвечает за определённую область.

## Структура рабочего пространства

```
prx-sd/
├── crates/
│   ├── cli/           # Бинарный файл "sd" — интерфейс командной строки
│   ├── core/          # Координация работы движка сканирования
│   ├── signatures/    # Хеш-БД (LMDB) + движок правил YARA-X
│   ├── parsers/       # Парсеры бинарных форматов
│   ├── heuristic/     # Эвристическая оценка + ML-инференс
│   ├── realtime/      # Мониторинг файловой системы + сетевая фильтрация
│   ├── quarantine/    # Зашифрованное карантинное хранилище
│   ├── remediation/   # Действия по реагированию на угрозы
│   ├── sandbox/       # Изоляция процессов + анализ поведения
│   ├── plugins/       # Среда выполнения плагинов WebAssembly
│   └── updater/       # Клиент обновления сигнатур
├── update-server/     # Сервер распространения сигнатур (Axum)
├── gui/               # Десктопный графический интерфейс (Tauri 2 + Vue 3)
├── drivers/           # Драйверы ядра ОС
│   └── windows-minifilter/  # Минифильтр файловой системы Windows (C)
├── signatures-db/     # Встроенные минимальные сигнатуры
├── packaging/         # Сборка дистрибутивов
├── tests/             # Интеграционные тесты
├── tools/             # Скрипты сборки и утилиты
├── install.sh         # Скрипт установки
└── uninstall.sh       # Скрипт удаления
```

## Конвейер обнаружения

Движок сканирования (`core`) координирует многоуровневый конвейер обнаружения:

```
                    ┌──────────────┐
                    │  Входной файл│
                    └──────┬───────┘
                           │
                    ┌──────▼───────┐
                    │  Магическое  │  Определение: PE, ELF, MachO,
                    │    число     │  PDF, ZIP, Office, неизвестный
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
       ┌──────▼──────┐    │     ┌──────▼──────┐
       │  Сопоставл.  │    │     │   YARA-X    │
       │    хешей     │    │     │  Правила    │
       │   (LMDB)    │    │     │  (38K+)     │
       └──────┬──────┘    │     └──────┬──────┘
              │     ┌─────▼─────┐      │
              │     │Эвристич.  │      │
              │     │  анализ   │      │
              │     └─────┬─────┘      │
              │           │            │
              │    ┌──────▼──────┐     │
              │    │ ML-инференс│     │
              │    │   (ONNX)   │     │
              │    └──────┬─────┘     │
              │           │            │
              │    ┌──────▼──────┐     │
              │    │ VirusTotal  │     │
              │    │Облачный зап.│     │
              │    └──────┬─────┘     │
              │           │            │
              └───────────┼────────────┘
                          │
                   ┌──────▼──────┐
                   │  Агрегация  │
                   │  вердикта   │
                   └─────────────┘
                   Clean / Suspicious / Malicious
```

### Детали уровней

1. **Сопоставление хешей** -- поиск за O(1) в базе LMDB, содержащей хеши SHA-256 и MD5 из ClamAV, abuse.ch, VirusShare и пользовательских блоклистов.

2. **Правила YARA-X** -- сопоставление по шаблонам с использованием движка YARA-X (нативная реализация YARA на Rust). Правила загружаются из встроенных наборов по умолчанию и внешнего репозитория сигнатур.

3. **Эвристический анализ** -- анализ, зависящий от типа файла:
   - **PE:** энтропия секций, подозрительные API-импорты (CreateRemoteThread, VirtualAllocEx), обнаружение упаковщиков (UPX, Themida), аномалии временных меток
   - **ELF:** энтропия секций, ссылки на LD_PRELOAD, закрепление через cron/systemd, шаблоны SSH-бэкдоров
   - **MachO:** энтропия секций, инъекция dylib, закрепление через LaunchAgent, доступ к Keychain

4. **ML-инференс** (опционально, флаг функции `onnx`) -- оценка ONNX-моделей через tract:
   - PE: 64-мерный вектор признаков (хеши таблицы импортов, энтропия секций, сигнатуры API)
   - ELF: 48-мерный вектор признаков (энтропия секций, таблица символов, динамические библиотеки)

5. **Облачный запрос VirusTotal** -- резервный вариант для файлов, не обнаруженных локально. Запрашивает API VirusTotal (бесплатный уровень: 500 запросов/день). Результаты кешируются в LMDB.

### Оценка

- Оценка >= 60: **Вредоносный (Malicious)**
- Оценка 30-59: **Подозрительный (Suspicious)**
- Оценка < 30: **Чистый (Clean)**

Итоговый вердикт определяется по наивысшему уровню угрозы среди всех уровней обнаружения.

## Защита в реальном времени

Крейт `realtime` обеспечивает непрерывную защиту через несколько подсистем:

| Подсистема | Linux | macOS | Windows |
|------------|-------|-------|---------|
| Мониторинг файлов | fanotify + epoll | FSEvents (notify) | ReadDirectoryChangesW (notify) |
| Перехват процессов | FAN_OPEN_EXEC_PERM | - | Minifilter (планируется) |
| Сканирование памяти | /proc/pid/mem | - | - |
| Обнаружение шифровальщиков | Мониторинг записи и переименования | Мониторинг записи и переименования | Мониторинг записи и переименования |
| Защищённые каталоги | ~/.ssh, /etc/shadow, /etc/systemd | ~/Library, /etc | Registry Run keys |
| DNS-фильтрация | Движок Adblock + списки IOC | Движок Adblock + списки IOC | Движок Adblock + списки IOC |
| Мониторинг поведения | /proc + audit (execve/connect/open) | - | - |

## Карантинное хранилище

Файлы помещаются в карантин с использованием аутентифицированного шифрования AES-256-GCM:

1. Генерация случайного 256-битного ключа + 96-битного nonce
2. Шифрование содержимого файла с помощью AES-256-GCM
3. Сохранение зашифрованного файла с именем в формате UUID
4. Сохранение метаданных в JSON (исходный путь, хеш, название угрозы, временная метка)
5. При восстановлении файл расшифровывается и проверяется целостность перед записью

## Конвейер устранения угроз

При использовании `--remediate`:

```
Обнаружена угроза
  ├── 1. Завершение процесса (SIGKILL на Linux/macOS, TerminateProcess на Windows)
  ├── 2. Карантин файла     (зашифрованное хранилище AES-256-GCM)
  └── 3. Очистка закрепления
        ├── Linux:   задания cron, сервисы systemd, LD_PRELOAD
        ├── macOS:   LaunchAgents, записи plist, Keychain
        └── Windows: реестр Run/RunOnce, запланированные задачи, службы
```

Действия настраиваются через `sd policy set`.

## База сигнатур

### Встроенные сигнатуры (`signatures-db/`)

Минимальный набор сигнатур, встроенный в бинарный файл `sd` через `include_str!`:
- Тестовая сигнатура EICAR
- Основные YARA-правила (шифровальщики, трояны, бэкдоры и т.д.)
- Хеши известных вредоносных программ (WannaCry, Emotet, NotPetya)

### Внешние сигнатуры ([prx-sd-signatures](https://github.com/openprx/prx-sd-signatures))

Полноценная, часто обновляемая база данных угроз:
- 38 800+ YARA-правил из 9 источников
- Хеш-блоклисты из лент abuse.ch
- Списки IOC: 585K+ вредоносных IP-адресов, доменов, URL

### Хранение

- **Хеши:** LMDB (крейт heed) для поиска ключ-значение за O(1)
- **YARA-правила:** загружаются и компилируются YARA-X при запуске
- **Списки IOC:** HashSet в памяти для быстрого сопоставления IP/доменов/URL

## Система плагинов

PRX-SD поддерживает плагины WebAssembly через Wasmtime:

- Плагины представляют собой файлы `.wasm` с манифестом (`plugin.json`)
- Поддержка WASI для доступа к файловой системе и окружению
- Реестр плагинов для обнаружения и загрузки
- Хост-функции, предоставляемые плагинам для доступа к результатам сканирования и конфигурации

## Система обновлений

Крейт `updater` и `update-server` обеспечивают безопасный конвейер обновлений:

1. Клиент проверяет наличие новых версий сигнатур на сервере обновлений
2. Сервер отвечает информацией о версии и URL для загрузки
3. Клиент загружает пакет сигнатур, сжатый с помощью zstd
4. Подпись пакета проверяется с помощью Ed25519 (ed25519-dalek)
5. Сигнатуры извлекаются и загружаются в LMDB

## Графическое приложение

Создано на базе Tauri 2 (Rust-бэкенд) + Vue 3 (TypeScript-фронтенд):

- Интеграция с системным треем и индикатором состояния
- Панель управления со статистикой угроз
- Сканирование файлов перетаскиванием
- Браузер карантина с функциями восстановления и удаления
- Управление мониторингом в реальном времени
- Настройки и конфигурация
- Многоязычная поддержка (10 языков)
